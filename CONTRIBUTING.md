# Contributing to ExoZora Core

## Contribution Rules
- No direct commits to `main`.
- All changes must be submitted via Pull Request.
- At least 1 approval is required before merge.
- Keep PRs small and scoped to a single concern.
- Architecture-impacting changes require an RFC in `docs/rfc/`.

## Development Expectations
- Follow repository structure and naming conventions.
- Update documentation alongside structural changes.
- Ensure formatting and checks pass before requesting review.
